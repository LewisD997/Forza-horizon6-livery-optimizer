# Non-Destructive Cleanup Candidate Planner

v0.6.2 adds a cleanup candidate planner.

It does not optimize geometry yet. It only writes `mark_candidate` entries into `optimization_plan.json` so future cleanup work can be reviewed before any shape is removed, updated, or replaced.

## What It Does

The planner looks for conservative signals such as:

- low-alpha large soft shapes
- tiny fragments inside dense local clusters
- small ellipse cluster members
- shapes overlapping high visual-difference regions
- background-like small shapes
- duplicate-like shapes

Every generated change is:

```text
action = mark_candidate
status = proposed
risk_level = review_only or low
```

No `remove_shape`, `update_shape`, or `add_shape` entries are generated by default.

## Candidate Types

Supported candidate types:

```text
low_alpha_large_soft_shape
tiny_fragment_cluster_member
ellipse_cluster_member
high_diff_overlap_member
background_or_near_background_shape
duplicate_like_shape
unknown_review_candidate
```

## Human Review

All candidates include:

```text
human_review_required = true
recommendation = review_before_cleanup
```

The planner treats `anime_artifact_analysis` and visual diff data as signals, not truth. Candidate entries are not instructions to delete layers.

## Run

```bash
python main.py --image cases/case_0001/source_full.png --input cases/case_0001/paintstudio_geometry.json --input-format paintstudio --report cases/case_0001/flo_report.json --output-geometry cases/case_0001/optimized_geometry.json --optimization-mode candidate-plan --plan-output cases/case_0001/optimization_plan.json --preview-renderer paintstudio-source --max-candidates 100
```

Useful options:

```text
--max-candidates 100
--min-candidate-score 0.35
--include-low-confidence
```

Validate:

```bash
python scripts/validate_optimization_plan.py --plan cases/case_0001/optimization_plan.json --geometry cases/case_0001/paintstudio_geometry.json
```

The optimized geometry output remains an unchanged copy in v0.6.2.
