# Candidate Review Visualization

v0.6.3 adds visualization tools for reviewing non-destructive cleanup candidates.

The input is an `optimization_plan.json` generated by candidate-plan mode. The output is a review folder with overlays, contact sheets, crops, a JSON index, a CSV table, and a text summary.

This is still non-destructive. Candidate visualization does not delete, update, add, or replace any Paint Studio shapes.

## Run

```bash
python scripts/render_candidate_review.py --case cases/case_0001 --top-n 50
```

Useful filters:

```bash
python scripts/render_candidate_review.py --case cases/case_0001 --candidate-type low_alpha_large_soft_shape
python scripts/render_candidate_review.py --case cases/case_0001 --candidate-type ellipse_cluster_member
python scripts/render_candidate_review.py --case cases/case_0001 --risk-level low
```

## Outputs

The script writes:

```text
cases/case_0001/candidate_review/candidate_overlay_all.png
cases/case_0001/candidate_review/candidate_overlay_low.png
cases/case_0001/candidate_review/candidate_overlay_review_only.png
cases/case_0001/candidate_review/candidate_contact_sheet_top_50.png
cases/case_0001/candidate_review/candidate_contact_sheet_low_alpha.png
cases/case_0001/candidate_review/candidate_contact_sheet_ellipse_cluster.png
cases/case_0001/candidate_review/candidate_review_index.json
cases/case_0001/candidate_review/candidate_review_table.csv
cases/case_0001/candidate_review/review_summary.txt
cases/case_0001/candidate_review/crops/
```

## How To Read It

- Overlay images show candidate regions with labels such as `C0001`.
- Contact sheets show thumbnails with candidate type, score, risk, and alpha.
- Crops are individual candidate views with bounding boxes.
- The CSV is for sorting and manual review.
- The JSON index summarizes counts and output paths.

## Risk Levels

`review_only` means the candidate is informational.

`low` means multiple conservative signals agreed, but it still does not mean the shape is automatically removable.

Every candidate must be manually reviewed before future cleanup.

## Candidate Types

Common candidate types:

- `low_alpha_large_soft_shape`
- `tiny_fragment_cluster_member`
- `ellipse_cluster_member`
- `high_diff_overlap_member`
- `background_or_near_background_shape`
- `duplicate_like_shape`
- `unknown_review_candidate`

These are review labels, not commands.
